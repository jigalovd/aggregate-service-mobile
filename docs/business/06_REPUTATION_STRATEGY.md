# Система Репутации и Отзывов (User Stories)

## E5: Reputation System (Расширенная версия)

### Описание
Система доверия между клиентами и мастерами. Отзывы привязаны к реальным услугам, имеют взвешенный рейтинг и защиту от накруток.

---

### User Stories (Детализированные)

#### US-5.1: Просмотр Рейтинга Мастера (Клиент)

**Как** Клиент
**Я хочу** видеть рейтинг мастера (звезды) и читать отзывы других
**Чтобы** принять решение о записи

**Критерии приемки:**
- [ ] Рейтинг показывается с точностью до 0.1 (например, 4.7)
- [ ] Отображается общее количество отзывов (например, "23 отзыва")
- [ ] Есть badge "Verified" для подтвержденных отзывов
- [ ] Распределение оценок (5 звезд: 18, 4 звезды: 3, ...)
- [ ] Отзывы сортируются по умолчанию: "сначала полезные" (взвешенные)

**UI:**
```
[★★★★★] 4.7 (23 отзыва)

Распределение:
5 ★★★★★ ████████████████████ 18
4 ★★★★☆  ████ 3
3 ★★★☆☆  █ 1
2 ★★☆☆☆  █ 1
1 ★☆☆☆☆  ░ 0
```

#### US-5.2: Оставление Отзыва (Клиент)

**Как** Клиент
**Я хочу** оставить отзыв только после завершенной услуги
**Чтобы** мои отзывы были честными и основанными на опыте

**Критерии приемки:**
- [ ] Кнопка "Оставить отзыв" доступна только если `booking.status = COMPLETED`
- [ ] Форма включает: рейтинг (1-5 звезд), текст (опционально), фото (опционально, future)
- [ ] Одному бронированию соответствует только один отзыв
- [ ] Отзыв можно редактировать в течение 24 часов
- [ ] После отправки отзывы НЕ удаляются (только редактируются)

**Validations:**
- [ ] API проверяет, что `booking.end_time` > 24 часа назад (anti-fraud: "слишком быстрый отзыв")
- [ ] Минимальная длина текста: 10 символов (если заполнено)
- [ ] Максимальная длина: 500 символов

**UI Flow:**
1. Клиент заходит в "Мои записи"
2. Видит запись со статусом "Выполнена"
3. Нажимает "Оставить отзыв"
4. Выбирает звезды ★★★★★
5. Пишет текст
6. Нажимает "Отправить"
7. Видит: "Спасибо! Ваш отзыв поможет другим"

#### US-5.3: Ответы на Отзывы (Мастер - Опционально)

**Как** Мастер
**Я хочу** иметь возможность ответить на отзыв клиента
**Чтобы** показать профессионализм и поблагодарить

**Критерии приемки:**
- [ ] Мастер видит все отзывы о себе в профиле
- [ ] Может оставить один ответ на каждый отзыв
- [ ] Ответы могут быть редактированы
- [ ] Ответы не влияют на рейтинг (только диалог)

**UI:**
```
Отзыв от Анны (14.09.2025):
★★★★★ "Отличный мастер!"

Ответ мастера:
Спасибо, Анна! Ждем вас снова! ❤️
[Изменить ответ]
```

**Priority:** Low (может быть добавлено в post-MVP)

#### US-5.4: Модерация Подозрительных Отзывов (Admin)

**Как** Администратор
**Я хочу** видеть queue подозрительных отзывов
**Чтобы** предотвратить накрутку рейтинга

**Критерии приемки:**
- [ ] Dashboard показывает отзывы с `is_suspicious = true`
- [ ] Для каждого отзыва отображается причина подозрения (IP clustering, flash voting, etc.)
- [ ] Admin может: Approve (подтвердить), Reject (удалить), Mark as OK (снять флаг)
- [ ] При действии пересчитывается рейтинг мастера

**Причины пометки `is_suspicious`:**
- [ ] Множественные отзывы от одного IP
- [ ] Отзыв через 1 минуту после завершения услуги
- [ ] Trust Score автора < 0.3
- [ ] Pattern "только негативные отзывы" у пользователя

**UI (Admin Panel):**
```
🚩 Подозрительные отзывы (45)

[Отзыв #123] IP Clustering
Автор: user@example.com (Trust: 0.2)
Мастер: Анна Макарова
Текст: "Лучшая мастер!" (5 звезд)
Причина: 3 отзыва с одного IP за неделю

[Approve] [Reject] [Details]
```

#### US-5.5: Trust Score (Скрытая Система)

**Как** Система
**Я хочу** присваивать каждому пользователю Trust Score
**Чтобы** взвешивать их голоса в рейтинге

**Критерии приемки:**
- [ ] Trust Score рассчитывается автоматически
- [ ] Пользователь НЕ видит свой Trust Score (скрыто)
- [ ] Отзывы от пользователей с низким Trust Score имеют меньший вес
- [ ] Trust Score обновляется каждые 24 часа

**Факторы (детали в `03_RATING_SYSTEM.md`):**
- Верифицированный телефон (+0.3)
- Завершенные бронирования (до +0.3)
- Возраст аккаунта (до +0.2)
- Подозрительные флаги (каждый -0.1 до -0.5)

#### US-5.6: Взвешенный Рейтинг (Система)

**Как** Система
**Я хочу** рассчитывать рейтинг мастера с учетом Trust Score авторов отзывов
**Чтобы** рейтинг был устойчив к накруткам

**Критерии приемки:**
- [ ] Формула: `Weighted Rating = Σ(rating × trust_score) / Σ(trust_score)`
- [ ] Рейтинг кешируется в `employees.rating_cached`
- [ ] Рейтинг обновляется при каждом новом отзыве
- [ ] История изменений хранится в `reputation_events`

**Пример:**
- Мастер получил 2 отзыва:
  - От верифицированного клиента (Trust 0.9): ★★★★★
  - От нового пользователя (Trust 0.3): ★☆☆☆☆
- **Взвешенный рейтинг:** (5×0.9 + 1×0.3) / (0.9+0.3) = 4.2 / 1.2 = 3.5
- **Обычный средний:** (5+1)/2 = 3.0

#### US-5.7: Детекция Flash Voting (Система)

**Как** Система
**Я хочу** автоматически детектировать всплески отзывов
**Чтобы** предотвратить накрутку рейтинга

**Критерии приемки:**
- [ ] Если мастер получает >5 отзывов за 24 часа → триггер
- [ ] Отзывы из всплеска помечаются `is_suspicious = true`
- [ ] Рейтинг мастера замораживается до ручной проверки
- [ ] Admin получает уведомление (email/push)

**Логика:**
```python
if daily_reviews_count > 5:
    mark_reviews_suspicious(employee_id)
    freeze_employee_rating(employee_id)
    notify_admin("Flash voting detected for employee {id}")
```

#### US-5.8: Детекция IP Clustering (Система)

**Как** Система
**Я хочу** детектировать множественные отзывы с одного IP
**Чтобы** выявить накрутку с одного устройства

**Критерии приемки:**
- [ ] Хранить `client_ip` для каждого отзыва
- [ ] Группировать отзывы по первым 3 октетам IP
- [ ] Если >3 отзывов от одного IP/подсети → пометить как подозрительные
- [ ] Понизить Trust Score всех участников кластера на -0.2

**Пример:**
```
IP: 192.168.1.XXX
- 192.168.1.10 → отзыв #1 (мастер Анна)
- 192.168.1.11 → отзыв #2 (мастер Анна)
- 192.168.1.12 → отзыв #3 (мастер Анна)
→ Триггер! Все 3 отзыва помечены подозрительными
```

**UI Flow:**

6. Видит: "Ваш отзыв обновлен"

## Нефункциональные Требования

### Производительность
- [ ] Рейтинг должен кешироваться (не пересчитываться при каждом запросе)
- [ ] Запрос списка отзывов: < 500ms для 20 элементов
- [ ] Trust Score пересчитывается фоновым job (каждые 24h)

### Безопасность
- [ ] Один отзыв = одна бронь (constraint на уровне БД)
- [ ] Отзыв нельзя удалить, только отредактировать (audit trail)
- [ ] Trust Score скрыт от пользователей

### Масштабируемость
- [ ] Система должна держать 10,000+ отзывов в день
- [ ] Fraud detection работает асинхронно (не блокирует UI)

## Success Metrics

### Метрики Запуска
- **CR (Conversion Rate)** на оставление отзыва: > 30% (от выполненных броней)
- **Средний рейтинг по платформе:** 4.2 - 4.8 (индикатор качества)
- **Процент подозрительных отзывов:** < 5%

### Метрики Качества
- **Retention мастеров:** > 80% (те, у кого рейтинг > 4.5)
- **Количество жалоб на unfair reviews:** < 1% от общего числа отзывов
