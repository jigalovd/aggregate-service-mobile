Документ содержит пользовательские истории для мобильного приложения Aggregate Service. Каждая история связана с эпиком и статусом реализации.

## Epic E1: Authentication (Аутентификация)

**Статус**: ✅ Complete (100%)

### US-1.1: Вход в систему

**Как** пользователь,
**Я хочу** войти в систему по email и паролю,
**Чтобы** получить доступ к персонализированным функциям приложения.

**Acceptance Criteria:**
- [x] Поле email с валидацией формата
- [x] Поле пароля с маскированием
- [x] Кнопка "Login" активна только при валидной форме
- [x] Отображение ошибки при неверных учётных данных
- [x] Индикатор загрузки во время запроса
- [x] Автоматический переход на главный экран после успешного логина

**Implementation**: [Auth Feature](../features/AUTH_FEATURE.md)

---

### US-1.2: Выход из системы

**Как** пользователь,
**Я хочу** выйти из системы,
**Чтобы** защитить свои данные на чужом устройстве.

**Acceptance Criteria:**
- [x] Кнопка выхода в профиле
- [x] Очистка локальных токенов
- [x] Переход на экран логина
- [x] Отзыв refresh token на сервере

**Implementation**: [Auth Feature](../features/AUTH_FEATURE.md)

---

### US-1.3: Автоматическое обновление токена

**Как** пользователь,
**Я хочу** оставаться в системе без повторного ввода пароля,
**Чтобы** не прерывать работу с приложением.

**Acceptance Criteria:**
- [x] Автоматический refresh при истечении access token
- [x] Прозрачный процесс для пользователя
- [x] Переход на экран логина при неудачном refresh

**Implementation**: [Auth Feature](../features/AUTH_FEATURE.md)
