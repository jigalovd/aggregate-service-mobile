# UX Guidelines - Aggregate Service

## Обзор

Базовые UX принципы и гайдлайны для мобильного приложения Aggregate Service.

---

## 🚀 Short User Flows: Anti-Mega-App Concept

### Ключевая философия

> **Цель:** Избежать ощущения "mega app" (перегруженного приложения). Фокус на speed и simplicity.

| Метрика                 | Текущее         | Целевое           | Улучшение        |
|-------------------------|-----------------|-------------------|------------------|
| **Client Booking**      | 7 шагов, ~3 мин | 2-3 шага, <60 сек | **-70% времени** |
| **Provider Daily Time** | 15-20 мин       | <5 мин            | **-75% времени** |
| **Taps per Action**     | 8-12            | 2-4               | **-66% taps**    |

## 📊 Архитектурные метрики эффективности

### Количественные цели интерфейса

| Метрика                  | Стандартное приложение | Целевой показатель | Улучшение        |
|--------------------------|------------------------|--------------------|------------------|
| **Client Booking**       | 7 шагов, ~3 мин        | 2-3 шага, <60 сек  | **-70% времени** |
| **Provider Daily Admin** | 15-20 мин              | <5 мин             | **-75% времени** |
| **Taps per Action**      | 8-12                   | 2-4                | **-66% taps**    |

### Формулы расчёта эффективности

```
Time Savings = (Standard_Time - Optimized_Time) / Standard_Time × 100%
Tap Reduction = (Standard_Taps - Optimized_Taps) / Standard_Taps × 100%
User Effort Score = Total_Interactions × Cognitive_Load_Per_Interaction
```

## 🚫 Anti-Mega-App Principles

### Принцип 1: "One Screen, One Purpose"

```
❌ MEGA-APP (плохо):
┌─────────────────────────────────────────────────────────────────┐
│  📅 Сегодняшние записи (3)  📊 Статистика  🔥 Горячие предложения│
│  📰 Новости  💬 Сообщения (5)  ⭐ Отзывы (2)  🎁 Бонусы  ...     │
└─────────────────────────────────────────────────────────────────┘

✅ SHORT FLOW (хорошо):
┌─────────────────────────────────────────────────────────────────┐
│  ⚡ БЫСТРАЯ ЗАПИСЬ (Одно действие на экране)                    │
│  [🔍 Найти мастера]  [❤️ Избранное]  [📅 Записи]               │
└─────────────────────────────────────────────────────────────────┘
```

#### Типографическая и пространственная дисциплина

Мобильные устройства накладывают жёсткие ограничения: ~1/5 площади десктопного монитора, значительная часть
перекрывается пальцами.

| Параметр                   | Минимальное значение | Обоснование                            |
|----------------------------|----------------------|----------------------------------------|
| **Шрифт основного текста** | 16px                 | Предотвращает напряжение глаз          |
| **Touch targets**          | 44×44 px             | Компенсирует неточность пальца         |
| **Рекомендуемые кнопки**   | 56×56 px             | Primary actions в зоне большого пальца |

**Когнитивное обоснование:** Пользователь должен мгновенно, на уровне периферийного зрения, считывать контекст и
понимать единственный правильный следующий шаг. Снижение общей сложности интерфейса важнее арифметического сокращения
шагов.

### Принцип 2: "Smart Defaults Over Configuration"

```
❌ MEGA-APP: Пользователь должен выбрать ВСЁ (дата, время, мастер, услуга, оплата)

✅ SHORT FLOW: Система предлагает:
   - Ближайшее доступное время у любимого мастера
   - One-tap подтверждение
   - Автоматическая оплата (сохраненная карта)
```

### Принцип 3: "Progressive Disclosure"

```
LEVEL 1 (всегда видно):
  [⚡ Быстрая запись]  [❤️ Избранное]  [📅 Записи]
           ↓ Pull-to-refresh
LEVEL 2 (по запросу):
  [📊 Статистика]  [💬 Сообщения]  [⭐ Отзывы]
           ↓ Настройки → Advanced
LEVEL 3 (скрыто):
  [⚙️ Настройки уведомлений]  [🔐 Безопасность]  [📦 Экспорт]
```

#### Типы прогрессивного раскрытия

| Тип                        | Механизм                                                             | Оптимальные сценарии                                                       |
|----------------------------|----------------------------------------------------------------------|----------------------------------------------------------------------------|
| **Conditional Disclosure** | Динамически отображает элементы при выполнении логических условий    | Сложные адаптивные формы, динамические фильтры, персонализированная оплата |
| **Contextual Disclosure**  | Изолирует релевантные детализированные данные для выбранного объекта | Карточки услуг, tooltips с ценообразованием, страницы профилей             |
| **Progressive Enabling**   | Деактивирует элементы до завершения предшествующего шага             | Аутентификация, юридические согласия, транзакции                           |
| **Staged Disclosure**      | Декомпозирует процесс на простые последовательные экраны             | Онбординг, многошаговые мастера настройки профиля                          |

**Важно:** Превышение 3 уровней раскрытия приводит к фрагментации ментальной модели и потере ориентации.

### Принцип 4: "Context Over Navigation"

```
❌ MEGA-APP: Главная → Профиль → Настройки → Уведомления → Push → Выключить (5 taps)

✅ SHORT FLOW: Long-press на уведомлении → [Выключить] (1 action)
```

### Принцип 5: "Swipe Over Tap"

```
❌ MEGA-APP: [Подтвердить] [Отклонить] [Перенести] [Подробнее] (4 кнопки)

✅ SHORT FLOW: ← [Отклонить]  Запись  [Подтвердить] → (swipe для действия)
```

#### Преимущества жестового управления

| Преимущество                | Описание                                                                          |
|-----------------------------|-----------------------------------------------------------------------------------|
| **Защита от ошибок**        | Свайп требует осознанного, продолжительного моторного усилия vs случайное касание |
| **Очистка интерфейса**      | Освобождённое пространство передаётся контенту                                    |
| **Повышение вовлечённости** | Прямое манипулирование объектами повышает чувство контроля                        |

#### ⚠️ Проблемы доступности жестов (WCAG 2.2)

| Проблема                 | Описание                                                            | Решение                               |
|--------------------------|---------------------------------------------------------------------|---------------------------------------|
| **Discoverability**      | Жесты невидимы на статичном экране                                  | Подсказки при первом использовании    |
| **Когнитивная нагрузка** | Сложные жесты разрушают ментальные модели                           | Максимум 2 базовых жеста (left/right) |
| **Моторные ограничения** | Пользователи с нарушениями мелкой моторики не могут выполнить свайп | Обязательный альтернативный метод     |

#### Принцип избыточности (Redundancy)

**Правило:** Каждый жест должен иметь альтернативный метод выполнения:

```
✅ СВАЙП ВПРАВО для подтверждения
   + АЛЬТЕРНАТИВА: Кнопка [✓ Подтвердить] при long-press

✅ СВАЙП ВЛЕВО для отклонения
   + АЛЬТЕРНАТИВА: Контекстное меню с кнопкой [✗ Отклонить]
```

**Реализация:**

```kotlin
// Всегда предоставлять альтернативу жесту
Box(
    modifier = Modifier
        .combinedClickable(
            onClick = { /* Альтернативное действие */ },
            onLongClick = { showContextMenu() }
        )
        .swipeable(/* Жест */)
)
```

### Принцип 6: "No Dead Ends"

```
❌ MEGA-APP: ✅ Запись подтверждена! [OK] ← Пользователь потерян

✅ SHORT FLOW: ✅ Запись подтверждена!
              📅 Добавить в календарь? [Да] [Нет]
              ❤️ Добавить в избранное? [Да] [Уже есть]
              [На главную] [Найти еще]
```

### Принцип 7: "Hide Advanced, Keep Simple"

```
❌ MEGA-APP: [📝 Создать] [📊 Аналитика] [👥 Клиенты] [💰 Финансы] [📸 Галерея]...

✅ SHORT FLOW: 📅 Сегодня (5 записей)
              [+ Новая запись]
              [≡ Меню] → (раскрыть остальное)
```

## 🔧 Микро-интеракции

### Модель микро-интеракции

Каждая микро-интеракция строится по алгоритмической модели из 4 элементов:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Триггер   │ ──► │   Правило   │ ──► │  Обратная   │ ──► │ Цикл/Режим  │
│             │     │   системы   │     │   связь     │     │             │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### Технические метрики

| Платформа   | Реализация                                                         | Tap-to-Response |
|-------------|--------------------------------------------------------------------|-----------------|
| **iOS**     | `UINotificationFeedbackGenerator().notificationOccurred(.success)` | ≤ 100ms         |
| **Android** | `HapticFeedbackConstants.LONG_PRESS`                               | ≤ 100ms         |

### Haptic Feedback (тактильное подтверждение)

```kotlin
// Пример: подтверждение бронирования
fun onBookingConfirmed() {
    HapticFeedback.mediumImpact()  // Вибрация средней силы
    showSuccessAnimation()
}

// Пример: ошибка
fun onBookingError() {
    HapticFeedback.notificationError()
    showErrorDialog()
}
```

### Типы тактильной обратной связи

| Тип         | Когда использовать             | Сила вибрации         |
|-------------|--------------------------------|-----------------------|
| **Light**   | Выбор элемента списка          | Лёгкая                |
| **Medium**  | Подтверждение действия         | Средняя               |
| **Heavy**   | Важное предупреждение          | Сильная               |
| **Success** | Успешное завершение транзакции | Специфический паттерн |
| **Error**   | Ошибка валидации/системы       | Специфический паттерн |

## 1. Общие принципы

### 1.1. Core Values

| Принцип           | Описание                           |
|-------------------|------------------------------------|
| **Простота**      | Минимум шагов для достижения цели  |
| **Скорость**      | Быстрый доступ к ключевым функциям |
| **Доверие**       | Прозрачность действий и информации |
| **Интуитивность** | Очевидный UI без обучения          |

### 1.2. Mobile-First

- **Thumb-friendly**: Кнопки в зоне большого пальца
- **One-handed**: Возможность работы одной рукой
- **Quick actions**: Частые действия на нижней панели

### 1.3. Key Differentiators (UVP)

| UVP                     | Описание                            | UX Implication              |
|-------------------------|-------------------------------------|-----------------------------|
| **Multi-Role System**   | Мастер может быть клиентом          | Role Switcher UI            |
| **Service-First Flow**  | Сначала услуги → потом мастер/время | Корзина услуг               |
| **Time-First Flow**     | "Здесь и сейчас" поиск              | Быстрый фильтр по времени   |
| **Transparent Pricing** | Цены upfront                        | Показывать цену в карточках |

## 3. UI Patterns для Short Flows

### 3.1. Bottom Sheet (Вместо отдельного экрана)

#### Спецификация

Bottom Sheet — элемент UI, закреплённый у нижней кромки экрана, динамически накладывающийся на текущий контент.

**Метрика:** +25-30% engagement vs традиционные модальные окна (сохранение spatial awareness).

#### Состояния Bottom Sheet

```
┌─────────────────────────────────────┐
│          FULL-SCREEN                │  ← Полностью развёрнут (90% экрана)
│    (Детальная информация)           │
├─────────────────────────────────────┤
```

| **Контент размыт под sheet** | Сохранение контекста предыдущего экрана                                |

## 5. Trade-offs (Что мы теряем)

### Гибкость vs Скорость

| Функция           | Mega-App                | Short Flow               | Trade-off           |
|-------------------|-------------------------|--------------------------|---------------------|
| **Выбор времени** | Любой слот              | Ближайшие 3-5            | Меньше контроля     |
| **Выбор мастера** | Полный список + фильтры | Избранное + рекомендации | Меньше discovery    |
| **Настройки**     | Все доступны            | Скрыты в меню            | Меньше кастомизации |

**Решение:** Progressive disclosure - базовые на виду, advanced в глубине.

### Детализация vs Простота

| Функция                   | Mega-App                | Short Flow               | Trade-off               |
|---------------------------|-------------------------|--------------------------|-------------------------|
| **Информация мастера**    | Полный профиль, галерея | Минимум (рейтинг, цена)  | Меньше инфо для решения |
| **Статистика провайдера** | Полная аналитика        | Только критичные метрики | Меньше бизнес-инсайтов  |

**Решение:** "Quick View" vs "Full Profile" - пользователь может углубиться.

## 6. Success Metrics

### Бизнес-метрики

| Метрика                       | Target (3 мес) | Target (6 мес) | Target (9 мес) |
|-------------------------------|----------------|----------------|----------------|
| **Booking completion time**   | <60 сек        | <45 сек        | <30 сек        |
| **Taps to book (repeat)**     | 2-3            | 2              | 1-2            |
| **Provider daily admin time** | <5 мин         | <3 мин         | <2 мин         |
| **Booking abandonment rate**  | <20%           | <15%           | <10%           |
| **User satisfaction (NPS)**   | >50            | >60            | >70            |

### Технические метрики UX

| Метрика                   | Target  | Измерение               |
|---------------------------|---------|-------------------------|
| **Tap-to-Response Time**  | ≤ 100ms | Haptic feedback latency |
| **Skeleton Loading Time** | < 200ms | Time to first skeleton  |
| **Screen Transition**     | 300ms   | EaseInOut animation     |
| **Modal Open**            | 250ms   | EaseOut animation       |
| **Button Press Feedback** | 100ms   | Ripple/haptic           |

### Формулы расчёта

```
Time Savings = (Standard_Time - Optimized_Time) / Standard_Time × 100%
Tap Reduction = (Standard_Taps - Optimized_Taps) / Standard_Taps × 100%
User Effort Score = Total_Interactions × Cognitive_Load_Per_Interaction
Booking Abandonment = (Started_Bookings - Completed_Bookings) / Started_Bookings × 100%
```

## 3.6. Multi-Role UI (Уникальная фича)

### Role Switcher

> **UVP:** Мастер может быть клиентом и записываться к другим мастерам.

**Ключевой принцип:** Обеспечение безошибочного понимания текущего статуса и прав в любой момент времени.

#### Визуальная дифференциация ролей

| Роль         | Цветовая схема                        | Психология                 |
|--------------|---------------------------------------|----------------------------|
| **Client**   | Холодные синие оттенки                | Спокойный, дружелюбный     |
| **Provider** | Энергичные оранжевые / строгие тёмные | Активный, профессиональный |

#### Компоненты Role Switcher

| Компонент                   | Назначение                                    |
|-----------------------------|-----------------------------------------------|
| **Dropdown**                | Быстрое переключение между ролями             |
| **Navigation Drawer**       | Список доступных профилей                     |
| **Аватар + статус**         | Визуальная идентификация текущей роли         |
| **Текстовый идентификатор** | Ясная подпись "Client Mode" / "Provider Mode" |

#### Правила дизайна

```
✅ ПРАВИЛЬНО:
┌─────────────────────────────────────────┐
│  ┌─────────┐                            │
│  │ Аватар  │  Мария Иванова             │
│  │         │  ▼ [Provider Mode]         │  ← Яркий индикатор роли
│  └─────────┘                            │
│                                         │
│  Текущая роль: PROVIDER                 │
│  • Управление расписанием               │
│  • Подтверждение записей                │
│                                         │
│  [Переключиться на Client]              │
└─────────────────────────────────────────┘

❌ НЕПРАВИЛЬНО:
┌─────────────────────────────────────────┐
│  Мария Иванова                          │  ← Неясно, какая роль активна
│  [Switch]                               │  ← Что переключается?
└─────────────────────────────────────────┘
```

#### Анимация переключения

```
Client → Provider:
1. Fade out Client UI (200ms)
2. "Переключение..." (300ms)
3. Fade in Provider Dashboard (200ms)
4. Обновление Bottom Nav иконок
5. Haptic feedback (medium)
```

### Role-Specific Navigation

| Role         | Bottom Nav                             | Primary Actions                |
|--------------|----------------------------------------|--------------------------------|
| **Client**   | Search, Bookings, Favorites, Profile   | Find master, Book service      |
| **Provider** | Schedule, Bookings, Analytics, Profile | Manage slots, Confirm bookings |

### Switch Animation

```
Client Mode → Provider Mode:
1. Fade out Client UI
2. Show "Switching to Provider..." (500ms)
3. Fade in Provider Dashboard
4. Update Bottom Nav icons
```

## 14. Checklist для UX Review (Updated for Short Flows)

### Перед релизом проверить:

#### Short Flows (NEW!)

- [ ] Client может записаться за <60 секунд (2-3 taps)
- [ ] Provider тратит <5 мин/день на управление
- [ ] Favorites-First главный экран реализован
- [ ] One-tap rebooking для повторных клиентов
- [ ] Swipe actions работают (confirm/reject/cancel)
- [ ] Bottom Sheets вместо отдельных экранов
- [ ] Inline actions (без переходов)

#### Anti-Mega-App Principles (NEW!)

- [ ] **One Screen, One Purpose** - нет перегруженных экранов
- [ ] **Smart Defaults** - система предлагает, не заставляет выбирать
- [ ] **Progressive Disclosure** - Level 1-2-3 функции разделены
- [ ] **Context Over Navigation** - long-press, swipe работают
- [ ] **Swipe Over Tap** - основные действия через swipe
- [ ] **No Dead Ends** - после каждого действия есть следующий шаг
- [ ] **Hide Advanced** - advanced функции в меню

#### Core UX

- [ ] Все кнопки имеют touch target >= 48dp
- [ ] Loading states для всех async операций
- [ ] Empty states для всех списков
- [ ] Error states для всех network запросов
- [ ] Pull to refresh для списков
- [ ] Confirmation dialogs для деструктивных действий

#### Service-First Flow

- [ ] Корзина услуг работает корректно
- [ ] Расчет времени и цены динамический
- [ ] Кнопки [Выбрать Мастера] и [Выбрать Время] видны
- [ ] Flow A (Master-First) работает
- [ ] Flow B (Time-First) работает
- [ ] Flow C (Direct Search) работает
- [ ] Flow D (One-Tap Rebooking) работает

#### Multi-Role System

- [ ] Role Switcher виден в Profile
- [ ] Переключение между ролями smooth
- [ ] Navigation меняется по роли
- [ ] Provider Dashboard доступен
- [ ] Client features доступны в Provider mode

#### Accessibility

- [ ] Accessibility labels для всех интерактивных элементов
- [ ] RTL layout работает корректно
- [ ] Color contrast соответствует WCAG 2.1

#### Internationalization

- [ ] Все тексты локализованы (ru, he, en)
- [ ] Даты форматируются по locale
- [ ] Валюты форматируются по locale
- [ ] RTL работает для Hebrew

#### Performance

- [ ] Animations не вызывают lag
- [ ] Images оптимизированы
- [ ] Lazy loading для списков
- [ ] Skeleton loading работает

#### Edge Cases

- [ ] Слот занят - показывается альтернатива
- [ ] Provider недоступен - fallback UI
- [ ] No internet - offline state
- [ ] Конфликт услуг - понятное сообщение

#### Forms & Validation

- [ ] Forms имеют валидацию
- [ ] Ошибки показываются inline
- [ ] Success feedback после submit
