## 🚨 CRITICAL: Business Rule Violations (MUST FIX)

### CRITICAL-1: Отсутствует проверка 2-часового окна для отмены бронирования

**User Story:** US-3.5 - "Я как Клиент хочу отменить запись (за 2 часа до начала)"

**Business Rule:** Клиент может отменить бронирование минимум за 2 часа до start_time. Если до начала осталось меньше 2 часов - отмена недоступна.

**File:** `feature/booking/src/commonMain/kotlin/com/aggregateservice/feature/booking/domain/usecase/CancelBookingUseCase.kt:28-40`

**Current Behavior:**
```kotlin
class CancelBookingUseCase(
    private val repository: BookingRepository,
) {
    suspend operator fun invoke(
        bookingId: String,
        reason: String? = null,
    ): Result<Booking> {
        // Validation: bookingId
        if (bookingId.isBlank()) {
            return Result.failure(
                AppError.ValidationError("bookingId", "Booking ID is required"),
            )
        }
        return repository.cancelBooking(bookingId, reason)
    }
}
```

**Expected Behavior:**
```kotlin
class CancelBookingUseCase(
    private val repository: BookingRepository,
) {
    suspend operator fun invoke(
        bookingId: String,
        reason: String? = null,
    ): Result<Booking> {
        if (bookingId.isBlank()) {
            return Result.failure(
                AppError.ValidationError("bookingId", "Booking ID is required"),
            )
        }

        // Получаем бронирование для проверки времени
        val booking = repository.getBookingById(bookingId).getOrNull()
            ?: return Result.failure(AppError.NotFound("Booking not found"))

        // Проверяем 2-часовое окно (US-3.5)
        val now = Instant.fromEpochMilliseconds(System.currentTimeMillis())
        val minCancelTime = Instant.fromEpochMilliseconds(
            booking.startTime.toEpochMilliseconds() - CANCEL_WINDOW_HOURS * 60 * 60 * 1000
        )

        if (now > minCancelTime) {
            return Result.failure(
                AppError.ValidationError(
                    "startTime",
                    "Cannot cancel booking less than $CANCEL_WINDOW_HOURS hours before start time"
                )
            )
        }

        return repository.cancelBooking(bookingId, reason)
    }

    companion object {
        private const val CANCEL_WINDOW_HOURS = 2L
    }
}
```

**Business Impact:**
- Мария (клиент) может отменить бронь за 5 минут до начала
- Анна (мастер) теряет время и потенциальный доход
- Нарушается справедливость для других клиентов

**Priority:** 🔴 BLOCKER

## ⚪ LOW: Recommendations

### LOW-1: BookingStatus transitions не формализованы

**Current State:** BookingStatus содержит все нужные статусы, но переходы между ними не формализованы в коде.

**File:** `feature/booking/src/commonMain/kotlin/com/aggregateservice/feature/booking/domain/model/BookingStatus.kt:13-19`

**Recommendation:** Добавить sealed class для валидации переходов:

```kotlin
enum class BookingStatus {
    PENDING,
    CONFIRMED,
    IN_PROGRESS,
    COMPLETED,
    CANCELLED,
    EXPIRED,
    NO_SHOW;

    fun canTransitionTo(newStatus: BookingStatus): Boolean {
        return when (this) {
            PENDING -> newStatus in listOf(CONFIRMED, CANCELLED, EXPIRED)
            CONFIRMED -> newStatus in listOf(IN_PROGRESS, CANCELLED, NO_SHOW)
            IN_PROGRESS -> newStatus == COMPLETED
            COMPLETED, CANCELLED, EXPIRED, NO_SHOW -> false
        }
    }
}
```

**Priority:** ⚪ LOW (backend контролирует переходы)
