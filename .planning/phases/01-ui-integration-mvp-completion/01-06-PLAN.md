  <action>
    Update AuthStateProviderImpl to expose currentUserId.

    Current implementation only has `isAuthenticatedFlow`:
    ```kotlin
    override val isAuthenticatedFlow: StateFlow<Boolean> =
        observeAuthStateUseCase()
            .map { it.isAuthenticated }
            .stateIn(...)
    ```

    Add a similar property for userId:
    ```kotlin
    override val currentUserId: String?
        get() = observeAuthStateUseCase().value.userId
    ```

    Note: We use `.value` (sync access) because:
    1. AuthState is already a StateFlow in ObserveAuthStateUseCase
    2. currentUserId is used in non-composable contexts where collectAsState isn't appropriate
    3. The AuthState object itself has userId as a property

    Alternatively, if we need reactive access:
    ```kotlin
    val currentUserIdFlow: StateFlow<String?> =
        observeAuthStateUseCase()
            .map { it.userId }
            .stateIn(
                scope = scope,
                started = SharingStarted.WhileSubscribed(5000),
                initialValue = null,
            )

    override val currentUserId: String?
        get() = currentUserIdFlow.value
    ```

    Use the reactive approach to stay consistent with the interface design.

    Add import for `map` if not already present.
  </action>
